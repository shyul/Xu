VirtualBench 19.0 Drops Support for Microsoft Windows 7 SP1, Windows 8.1, Windows Server 2012 R2, and Windows Server 2008 R2 SP1

With this release, VirtualBench drops support for Microsoft Windows 7 RTM SP1, Windows 8.1, Windows Server 2012 R2 and Windows Server 2008 R2 SP1. VirtualBench 19.0 and later will not install or run on an unsupported OS. You cannot deploy or distribute applications that use VirtualBench 19.0 to an unsupported OS. Additionally, after installing VirtualBench 19.0, you cannot use any installers built on this computer with any version of LabVIEW, LabWindows™/CVI™, NI TestStand™, or Measurement Studio on an unsupported OS.

Automating the Installation of NI Products

You can automate the installation of most NI products using command-line arguments to suppress some or all of the installer user interface and dialog boxes.

If the NI product you are installing uses Microsoft .NET 4.6.2, the .NET installer may run before any NI software installs and may require a reboot before the installation of NI software begins. To avoid a .NET reboot, install .NET 4.6.2 separately before you install NI software.

For more information about automating the installation of NI products, refer to Automating an Installer in the NI Package Manager manual.

Using NI Software with Microsoft Windows 10

Microsoft Windows 10 is the latest version of the Windows operating system and features significant changes compared to previous versions. Windows 10 introduces several new capabilities and also combines features from both Windows 7 and Windows 8. For more information about NI support for Windows 10, visit ni.com/windows10.
